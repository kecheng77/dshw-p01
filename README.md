## 数据来源

- **股票行情**：akshare（`ak.stock_zh_a_hist()`），后复权，日度
- **市场指数**：
  - 沪深300（000300）：akshare（`ak.index_zh_a_hist()`），CAPM分析的市场基准（必选）
  - 中证500（000905）：akshare（`ak.index_zh_a_hist()`），反映中小盘股表现，与沪深300形成互补，选择理由是它代表了与大盘股风格不同的市场维度
- **宏观指标**：
  - CPI同比增速：来源 akshare（`ak.macro_china_cpi_yearly()`），衡量通胀水平
  - M2同比增速：来源 akshare（`ak.macro_china_money_supply()`），选择理由：M2增速反映市场流动性充裕程度，直接影响股市资金面和估值水平
- **财务数据**：akshare（`ak.stock_financial_analysis_indicator()`），获取ROE和净利润率

## 存储方式

- **基础**：CSV（方式 A）— 所有原始数据和清洗后数据均以CSV存储
- **进阶**：SQLite（方式 C）— 将清洗后数据存入 `data/combined/fin_data.db`
- **选择SQLite的理由**：
  1. SQLite支持SQL查询，可高效进行跨表JOIN操作，将股票行情与宏观数据按月份关联
  2. 单文件数据库，无需安装数据库服务，便于项目分发
  3. 相比CSV，SQLite有事务支持、索引加速、类型约束等优势
  4. 金融数据天然适合关系型存储（股票信息表、行情表、宏观数据表之间有关联关系）

### CSV格式的优缺点

- **优点**：通用性极强，任何工具和语言都可读取；人类可读；简单直观
- **力不从心的场景**：数据量超过GB级别时读写缓慢；不支持列式读取（无法只加载部分列）；无类型约束和索引；并发写入易冲突

## 如何运行

1. 安装依赖：`pip install -r requirements.txt`
2. 运行 `01_download.ipynb` 下载原始数据
3. 运行 `02_clean.ipynb` 清洗并存储数据
4. 运行 `03_analysis.ipynb` 查看分析结果
5. 打开 `report.html` 阅读完整报告
